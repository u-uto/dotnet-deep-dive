# 02 Reference Types

## 概要

参照型は、**実体そのものではなく、その実体への参照を持つ型**です。  
C# では、オブジェクトの共有、null の扱い、メソッド間の影響範囲、メモリ上の振る舞いを理解するうえで重要です。

---

## 定義

参照型とは、変数の中に実体そのものではなく、実体を指す参照を持つ型です。  
別の変数に代入すると、参照がコピーされます。

代表的な参照型には次のようなものがあります。

- `class`
- `string`
- 配列
- `object`
- `delegate`
- `interface`

---

## 特徴

### 1. 代入で参照がコピーされる

参照型は、別の変数に代入したときに参照がコピーされます。  
そのため、同じ実体を複数の変数から共有できます。

```csharp
var a = new List<int> { 1, 2, 3 };
var b = a;

b.Add(4);
```

この場合、a と b は同じ List<int> を参照しているため、a 側にも 4 が反映されます。

### 2. 変更が共有される

参照型は、同じ実体を複数箇所で扱えるため、変更が共有されます。
この性質は便利ですが、意図しない副作用の原因にもなります。

### 3. null を持てる

参照型は null を持つことができます。
ただし、null を許容するかどうかは設計上の重要な判断です。




### 4. 同じ実体かどうかと、同じ値かどうかを分けて考える必要がある

参照型では、「同じオブジェクトを指しているか」と「中身が同じか」は別問題です。
この違いを理解していないと、比較や変更の挙動で混乱しやすくなります。

### 5. メモリ共有の影響を受けやすい

参照型は共有されるため、ある箇所での変更が別の箇所に影響する可能性があります。
設計段階で責務を分けることが重要です。

---

## 代表例

class

最も基本的な参照型です。

```csharp
public class Person
{
    public string Name { get; set; }
}
```

string

文字列を表す参照型です。
見た目は値のように扱われますが、内部的には参照型です。

```csharp
string message = "Hello";
```

配列

配列も参照型です。

```csharp
int[] numbers = { 1, 2, 3 };
```

object

すべての型の基底となる型です。

```csharp
Func<int, int> doubler = x => x * 2;
```

delegate

メソッドへの参照を扱います。

```csharp
object value = 123;
```

interface

振る舞いの契約を表します。

```csharp
public interface ILogger
{
    void Log(string message);
}
```

---

## 使いどころ

参照型は、次のような場面で向いています。
- 同じ実体を複数箇所で共有したい
- 変更を共有したい
- 複雑な状態を持たせたい
- ライフサイクルを管理したい
- 振る舞いを抽象化したい

たとえば、サービス、リポジトリ、コントローラ、ドメインオブジェクト、コレクションなどは参照型と相性がよいです。

逆に、単純な値を表したいだけで共有が不要なら、参照型を使う必然性は弱くなります。

---

## 注意点

### 1. 代入しても複製ではない

参照型は、代入しても実体が複製されるわけではありません。
参照が共有されるだけなので、変更は連動します。

### 2. 意図しない副作用が起きやすい

同じインスタンスを複数箇所で触ると、予想外の箇所に影響が出ることがあります。
状態を持つオブジェクトほど注意が必要です。

### 3. null を許容しすぎると設計が崩れる

参照型は null を持てますが、何でも null にすると、null チェックが散らばりやすくなります。
null を使うかどうかは設計で決めるべきです。

### 4. 参照の同一性と値の等価性を混同しやすい

「同じインスタンスか」と「同じ内容か」は別です。
比較の基準を誤ると、バグの原因になります。

### 5. 共有状態はテストしづらくなる

共有されるオブジェクトは、状態変化の追跡が難しくなります。
テスト可能性を意識して設計する必要があります。

## 実装例

### 参照の共有

```csharp
var a = new List<int> { 1, 2, 3 };
var b = a;

b.Add(4);

Console.WriteLine(string.Join(", ", a)); // 1, 2, 3, 4
Console.WriteLine(string.Join(", ", b)); // 1, 2, 3, 4
```

### class の代入

```csharp
public class Person
{
    public string Name { get; set; } = string.Empty;
}

var p1 = new Person { Name = "Alice" };
var p2 = p1;

p2.Name = "Bob";

Console.WriteLine(p1.Name); // Bob
Console.WriteLine(p2.Name); // Bob
```

### string の扱い

```csharp
string a = "Hello";
string b = a;

b = "World";

Console.WriteLine(a); // Hello
Console.WriteLine(b); // World
```

string は参照型ですが、不変です。
そのため、見た目は値型のように扱いやすい特徴があります。

### null を持つ参照型

```csharp
string? name = null;

if (name is null)
{
    Console.WriteLine("name is null");
}
```

### メソッドに渡した場合

```csharp
static void Rename(Person person)
{
    person.Name = "Changed";
}

var person = new Person { Name = "Original" };
Rename(person);

Console.WriteLine(person.Name); // Changed
```

## 設計指針

参照型を設計するときは、次の点を意識します。
- 共有したい状態かどうかを明確にする
- 変更がどこへ伝播するかを意識する
- null を許容するかどうかを決める
- 不変にできるなら不変にする
- 大きな責務を1つのオブジェクトに詰め込みすぎない
- 比較基準を明確にする

特に、状態を持つ参照型は便利ですが、扱いを誤るとバグの温床になります。
「誰が変更できるか」を設計で明確にしておくことが重要です。

---

## 判断基準

次の問いに答えると、参照型を選ぶべきか判断しやすくなります。
- 同じ実体を共有したいか
- 変更を複数箇所に反映したいか
- 状態を持たせたいか
- 参照の同一性が重要か
- null を許容する必要があるか
- コピーよりも共有のほうが自然か

これらに「はい」と言えるなら、参照型は有力な選択肢です。
逆に、単純な値として扱いたいなら値型を検討します。

---

まとめ

参照型は、実体そのものではなく参照を持つ型です。
コピーではなく共有が起きること、null を持てること、副作用が出やすいことが特徴です。

参照型を正しく理解すると、オブジェクト共有、状態管理、メソッドの影響範囲、設計の責務分離が安定します。
C# の実務設計で非常に重要な概念です。
