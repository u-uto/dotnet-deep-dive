# 04_variables  ：変数

## 概要

このファイルでは、C# の **変数** を整理します。  
変数は、値を入れておくための基本的な入れ物です。  
プログラムの多くは、変数を使ってデータを受け取り、加工し、次の処理へ渡します。

変数を理解すると、次のようなことが見えてきます。

- 値をどこに持っておくか
- その値を再利用できるか
- 型をどう指定するか
- `var` をどう見るか
- 代入と再代入の違い
- 変数名をどう付けるか

このファイルでは、C# の文法としての変数だけでなく、**実務で読みやすい変数の扱い方** まで整理します。

---

## 定義

### 変数とは何か

**変数** は、値を保持するための名前付きの領域です。  
C# では、変数を宣言してから値を入れ、必要に応じて参照します。

例:

```csharp
int age = 20;
string name = "Taro";
```

この場合、`age` と `name` が変数です。  
それぞれに型と値が紐づいています。

### 変数宣言とは何か

**変数宣言** は、変数の名前と型を決めることです。  
必要に応じて、宣言と同時に初期値も与えます。

例:

```csharp
int count;
count = 10;
```

### 代入とは何か

**代入** は、変数に値を入れることです。  
C# では `=` を使います。

例:

```csharp
count = 20;
```

---

## 特徴

### 1. 型と一緒に使う
C# は静的型付け言語なので、変数には型があります。  
そのため、どんな値でも入れられるわけではありません。

### 2. 何度も使える
変数は、一度値を入れたあと、同じ値を何度も参照できます。  
処理の途中結果を保持するのに向いています。

### 3. 再代入できる
多くの変数は、あとから別の値を代入できます。  
ただし、`const` ではなく通常の変数の場合です。

### 4. `var` で型を省略できる
C# では、型を明示せずに `var` を使うことができます。  
ただし、実際には型がなくなるわけではなく、**コンパイル時に型が決まる** だけです。

### 5. 可読性に影響する
変数名や宣言方法が悪いと、処理全体が読みにくくなります。  
変数は小さな要素ですが、コードの見やすさへの影響は大きいです。

---

## 代表例

### 1. 明示的に型を書く

```csharp
int age = 20;
string userName = "Taro";
decimal price = 1200m;
```

これは最も分かりやすい基本形です。  
型と値がはっきり見えます。

---

### 2. `var` を使う

```csharp
var age = 20;
var userName = "Taro";
var price = 1200m;
```

`var` を使っても、実際には型は決まっています。  
この例では `age` は `int`、`userName` は `string`、`price` は `decimal` です。

---

### 3. 宣言後に代入する

```csharp
int count;
count = 10;
```

変数は先に宣言し、あとから値を入れることもできます。  
ただし、実務では初期化までまとめて書くほうが分かりやすいことが多いです。

---

### 4. 再代入する

```csharp
int score = 50;
score = 80;
```

変数は、必要に応じて別の値に更新できます。  
状態が変わる処理ではよく使います。

---

## 使いどころ

変数は、ほぼすべての C# コードで使います。  
特に次の場面で重要です。

- 入力値を保持するとき
- 計算結果を一時的に保存するとき
- 条件分岐の判定値を持たせるとき
- ループのカウンタを扱うとき
- オブジェクトや中間結果を参照するとき

変数は、プログラムの「考えた結果を置いておく場所」として働きます。

---

## 注意点

### 1. 変数名が曖昧だと読みづらい
`x` や `temp` のような名前は、短くても意味が伝わりにくいことがあります。  
役割が分かる名前を付けるほうがよいです。

### 2. 初期化し忘れに注意する
宣言しただけで値を入れていない変数は、使えない場合があります。  
C# では、未初期化の変数をそのまま参照するとエラーになります。

### 3. `var` を何でも使えばよいわけではない
`var` は便利ですが、見ただけで型が分かりにくい場面では逆効果になることがあります。  
特に複雑な式では、明示的な型のほうが読みやすいことがあります。

### 4. 変数の寿命を短くする
必要以上に長く生かさず、使う場所の近くで宣言するほうが読みやすいです。  
変数の範囲が広すぎると、状態追跡が難しくなります。

## 実装例

### 例1: 基本的な変数宣言

```csharp
int age = 20;
string name = "Taro";
bool isActive = true;
```

この形は、変数の型と値をひと目で確認しやすいです。

---

### 例2: `var` を使う例

```csharp
var count = 100;
var message = "Hello";
var rate = 1.5;
```

`var` は、右辺から型が明らかなときに便利です。

---

### 例3: 変数を更新する例

```csharp
int total = 0;
total = total + 10;
total += 5;
```

変数は、計算結果を蓄積していく用途に向いています。

---

### 例4: 条件判定に使う例

```csharp
int score = 75;
bool isPass = score >= 60;
```

変数は、途中結果を別名で持つことで処理を読みやすくします。

## 設計指針

### 1. 変数は目的を持たせる
何を表しているかが分かる変数名にします。

### 2. 宣言と初期化を近づける
使う直前で宣言するほうが、追跡しやすくなります。

### 3. 不必要に再代入しない
値が変わらないなら、できるだけ変更しない設計のほうが安全です。  
必要に応じて後続の `constants` 章ともつながります。

### 4. `var` と明示型を使い分ける
読みやすさを基準に選びます。  
右辺を見なくても型が分かるなら `var` は使いやすいです。

---

## 判断基準

変数をどう書くかは、次の基準で判断します。

### 1. その変数は何を表すか
意味が明確であることが最優先です。

### 2. その場で型が分かるか
分かるなら `var` でもよいです。  
分かりにくいなら明示型がよいです。

### 3. 値は変わるか
変わらないなら、定数や readonly のほうが適切な場合があります。

### 4. 変数の生存範囲は適切か
広すぎると読みにくく、狭すぎると扱いづらくなります。

### 5. 他の変数と役割が重複していないか
似た変数が増えすぎると、コードの流れが見えにくくなります。

---

## よくある誤解

### 1. `var` は型なし
`var` は型をなくす機能ではありません。  
コンパイル時に型が決まります。

### 2. 変数は何度でも自由に変えてよい
何度も変えると追いづらくなります。  
必要最小限の再代入にしたほうが読みやすいです。

### 3. 短い名前のほうが速く書けてよい
書くのは早くても、読むときに遅くなります。  
実務では読む時間のほうが長いです。

### 4. 宣言はどこにあっても同じ
実際には、宣言位置で読みやすさが変わります。  
使う場所に近いほうが理解しやすいです。

---

## まとめ

変数は、C# の基本中の基本です。  
値を一時的に保持し、再利用し、処理の流れを支える役割を持っています。

このファイルで押さえるべき点は次のとおりです。

- 変数は値を保持する名前付きの領域
- C# では型とセットで扱う
- `var` は型を消すものではない
- 再代入は便利だが、乱用すると読みにくい
- 役割が分かる名前を付けることが重要
