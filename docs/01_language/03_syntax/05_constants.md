# 05_constants  ：定数

## 概要

このファイルでは、C# の **定数** を整理します。  
定数は、基本的に **変わらない値** を表すために使います。

変数が「値を保持して必要に応じて変えるもの」だとすると、定数は「最初から最後まで同じであることを前提にするもの」です。  
この違いを理解すると、コードの意図がかなり明確になります。

このファイルで扱う内容は次のとおりです。

- 定数の意味
- `const` の使い方
- `readonly` との違い
- どんな値を定数にするか
- 実務での使いどころ
- 定数化しすぎるときの注意点

---

## 定義

### 定数とは何か

**定数** は、変更されないことを前提とした値です。  
C# では、主に `const` を使って定義します。

例:

```csharp
const double Pi = 3.14159;
```

この場合、`Pi` は定数であり、あとから別の値を代入することはできません。

### `const` とは何か

`const` は、コンパイル時に確定する定数を表すキーワードです。  
`const` で定義した値は、実行中に変更できません。

例:

```csharp
const int MaxRetryCount = 3;
```

---

## 特徴

### 1. 変更されないことを明示できる
定数を使うと、「この値は変わらない」という意図をコード上で示せます。  
これにより、読む人が安心して扱えます。

### 2. 意味のある名前を付けやすい
数値や文字列をそのまま書く代わりに、意味のある名前を付けられます。  
これにより、マジックナンバーやマジックストリングを減らせます。

### 3. `const` はコンパイル時定数
`const` は、コンパイル時に値が確定している必要があります。  
そのため、実行時に決まる値には使えません。

### 4. 読みやすさと保守性が上がる
同じ値を何度も直接書くより、定数にまとめたほうが変更しやすくなります。

---

## 代表例

### 1. 数値の定数

```csharp
const int MaxItems = 100;
const int DefaultTimeoutSeconds = 30;
```

### 2. 文字列の定数

```csharp
const string AppName = "DotnetDeepDive";
const string ErrorMessage = "An error occurred.";
```

### 3. 小数の定数

```csharp
const decimal TaxRate = 0.1m;
const double Pi = 3.1415926535;
```

### 4. 論理値の定数

```csharp
const bool IsEnabled = true;
```

---

## 使いどころ

定数は、次のような場面でよく使います。

- 変更しない前提の設定値
- しきい値
- 固定メッセージ
- 固定の数値
- アプリケーション全体で共通の値
- 比較や判定で使う決まった値

たとえば、最大件数や税率、既定メッセージなどは定数にしやすいです。

---

## 注意点

### 1. なんでも定数にしない
一見固定に見えても、将来変わる可能性があるものまで無理に定数化すると、かえって扱いづらくなります。  
本当に変わらないものだけを定数にします。

### 2. `const` は実行時に決まる値には使えない
環境変数や設定ファイルから読む値は、`const` にはできません。  
その場合は変数や `readonly` を検討します。

### 3. 定数名は意味が分かるようにする
`A` や `X` のような名前では、何の値か分かりません。  
用途が伝わる名前にします。

### 4. 文字列定数を増やしすぎない
固定文字列を定数化しすぎると、逆に追いにくくなることがあります。  
再利用性が高いものだけを定数にするのがよいです。

---

## 実装例

### 例1: 数値定数

```csharp
const int MaxRetryCount = 3;
const int PageSize = 20;
```

### 例2: 文字列定数

```csharp
const string ApplicationName = "DemoApp";
const string DefaultUserName = "Guest";
```

### 例3: 定数を使った計算

```csharp
const decimal TaxRate = 0.1m;

decimal subtotal = 1000m;
decimal total = subtotal + (subtotal * TaxRate);
```

### 例4: 判定に使う定数

```csharp
const int AdultAge = 18;

int age = 20;
bool isAdult = age >= AdultAge;
```

## `readonly` との違い

定数を扱うときに、`readonly` との違いも重要です。

### `const`
- コンパイル時に値が確定する
- 実行時に変更できない
- 基本的に固定値向け

### `readonly`
- 宣言時またはコンストラクタ内でのみ代入できる
- 実行時に決まる値にも使える
- オブジェクトや設定値にも使いやすい

例:

```csharp
const int MaxRetryCount = 3;
readonly string _connectionString;
```

`readonly` は、**完全な定数ではないが、変更させたくない値** に向いています。  
この違いは後で `types/` や `oop/` に進むとさらに重要になります。

---

## 設計指針

### 1. 変わらないものだけを定数にする
定数は「仕様として固定されているもの」に向けます。

### 2. マジックナンバーを減らす
コード中に直接書かれた意味のない数値を定数へ置き換えます。

### 3. 共通値は一箇所に集める
同じ値を複数箇所で使うなら、定数化すると変更しやすくなります。

### 4. `const` と `readonly` を使い分ける
コンパイル時に決まるなら `const`、実行時に決まるなら `readonly` を考えます。

---

## 判断基準

定数にするかどうかは、次の基準で判断します。

### 1. その値は本当に変わらないか
変わる可能性があるなら、定数にしないほうがよいです。

### 2. 複数箇所で使うか
複数箇所で使うなら、定数化する価値が高いです。

### 3. 意味を持たせたいか
そのままの値より、名前を付けたほうが意図が伝わるなら定数向きです。

### 4. 実行時に必要な値か
実行時に決まる値なら `const` は使えません。

---

## よくある誤解

### 1. 定数はただの読み取り専用変数
`const` は単なる読み取り専用ではありません。  
コンパイル時に確定する特別な値です。

### 2. なんでも定数にすればよい
定数化しすぎると、逆に柔軟性が下がります。  
固定する意味があるものだけに使います。

### 3. `readonly` と同じ
`const` と `readonly` は似ていますが、使える場面が違います。  
同じものとして扱わないほうがよいです。

### 4. 定数は数値だけ
文字列や論理値にも使えます。  
用途は数値に限りません。

---

## まとめ

定数は、変わらない値を明示するための重要な要素です。  
`const` を使うことで、コードの意図をはっきりさせ、保守性を高められます。

このファイルで押さえるべき点は次のとおりです。

- 定数は変更されない値を表す
- `const` はコンパイル時定数
- `readonly` とは役割が違う
- マジックナンバーやマジックストリングを減らせる
- 変わらないものだけを定数化するのが基本
