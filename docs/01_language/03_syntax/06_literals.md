# 06_literals

## 概要

このファイルでは、C# の **リテラル** を整理します。  
リテラルとは、コードの中に直接書かれる値のことです。

たとえば、`10`、`"Hello"`、`true`、`null` のような値は、いずれもリテラルです。  
変数や定数に入れる前の「生の値」と考えると分かりやすいです。

このファイルで扱う内容は次のとおりです。

- リテラルの意味
- 数値リテラル
- 文字リテラル
- 文字列リテラル
- 論理値リテラル
- `null`
- 接尾辞や記法の違い
- 実務での注意点

---

## 定義

### リテラルとは何か

**リテラル** は、ソースコードに直接書かれた値です。  
変数のように名前を持たず、そのまま値そのものを表します。

例:

```csharp
int age = 20;
string name = "Taro";
bool isActive = true;
```

この場合、`20`、`"Taro"`、`true` がリテラルです。

### リテラルの役割

リテラルは、プログラムに具体的な値を与えるために使います。  
計算、比較、初期化、判定など、ほとんどの処理の基礎になります。

---

## 特徴

### 1. 値をそのまま書ける
リテラルは、直接値を表せるため、コードの初期化や簡単な処理でよく使われます。

### 2. 型によって書き方が変わる
数値、文字、文字列、論理値、null など、種類ごとに表現方法が違います。  
同じ「値」でも、書き方には違いがあります。

### 3. 型推論や型変換に影響する
リテラルの種類によって、C# がどの型として扱うかが変わります。  
たとえば、`10` は通常 `int` として扱われますが、`10L` は `long` です。

### 4. 読みやすさに直結する
同じ値でも、書き方が不自然だと読みにくくなります。  
特に文字列や日付の表現は、見た目の分かりやすさが重要です。

---

## 代表例

### 1. 数値リテラル

```csharp
int count = 10;
long total = 100L;
float rate = 1.5f;
double price = 1200.75;
decimal tax = 0.1m;
```

### 2. 文字リテラル

```csharp
char firstLetter = 'A';
char newline = '\n';
```

### 3. 文字列リテラル

```csharp
string message = "Hello";
string path = @"C:\temp\sample.txt";
string userName = "Taro";
```

### 4. 論理値リテラル

```csharp
bool isActive = true;
bool isDeleted = false;
```

### 5. null

```csharp
string? name = null;
object? value = null;
```

---

## 使いどころ

リテラルは、次の場面で特に重要です。

- 初期値を入れるとき
- 固定値をコードに書くとき
- 条件分岐の比較対象にするとき
- 文字列や数値を直接扱うとき
- 型の違いを確認するとき

リテラルはごく基本的ですが、コード全体で頻繁に使われます。  
そのため、書き方を理解しておくとミスが減ります。

---

## 注意点

### 1. マジックナンバーを増やしすぎない
意味のない数値をそのまま書くと、後で意図が分かりにくくなります。  
よく使う値は定数に置き換えるほうがよいです。

### 2. 数値の型を意識する
`int`、`long`、`float`、`double`、`decimal` では扱いが違います。  
特に `decimal` は金額などで重要です。

### 3. 文字列の書き方を使い分ける
普通の文字列、逐語的文字列、補間文字列など、用途によって見やすさが変わります。

### 4. `null` の扱いに注意する
`null` は「値がない」ことを意味します。  
参照型や nullable 型と関係が深く、以降の `nullable-types` 章にもつながります。

### 5. 文字と文字列を混同しない
`'A'` は文字、`"A"` は文字列です。  
見た目が似ていますが、まったく別の型です。

## 実装例

### 例1: 数値リテラル

```csharp
int age = 20;
long population = 8_000_000L;
decimal price = 1980.50m;
```

`_` を使うと、大きな数値を読みやすくできます。

---

### 例2: 文字と文字列

```csharp
char initial = 'A';
string name = "Alice";
```

`char` は 1 文字、`string` は複数文字を表します。

---

### 例3: 逐語的文字列

```csharp
string path = @"C:\temp\sample.txt";
```

`@"..."` の形を使うと、バックスラッシュをそのまま書けます。

---

### 例4: 文字列補間

```csharp
string userName = "Taro";
string message = $"Hello, {userName}";
```

補間文字列は、値を文字列の中に自然に埋め込めるため、実務でよく使います。

---

### 例5: null

```csharp
string? middleName = null;
```

`null` を使うと、まだ値がない状態を明示できます。

## 設計指針

### 1. リテラルは意味のある場面だけで使う
明らかな初期値や比較値にはそのまま使えます。  
ただし、繰り返し出る値は定数化を検討します。

### 2. 型が分かる書き方を意識する
リテラルは値だけでなく、型の理解にもつながります。  
特に数値系は接尾辞を意識します。

### 3. 読みやすい表現を選ぶ
見た目が複雑になる場合は、書き方を変えるだけでかなり見やすくなります。

### 4. `null` を雑に使わない
`null` は便利ですが、設計上の意図がないまま使うと扱いづらくなります。  
必要性を意識して使います。

---

## 判断基準

リテラルをどう書くかは、次の基準で判断します。

### 1. その値は何を表しているか
意味が明確な値ならそのまま使えます。

### 2. 型は明らかか
型が分かりにくいときは、接尾辞や明示的な型を使います。

### 3. 再利用されるか
何度も使うなら定数にしたほうがよい場合があります。

### 4. 読みやすいか
数値や文字列が見やすく書けているかを確認します。

---

## よくある誤解

### 1. リテラルはただの数字や文字
リテラルは単なる記号ではなく、型と意味を持つ値です。

### 2. どの数値も同じ
`int`、`long`、`double`、`decimal` では扱いが違います。  
用途に応じて選ぶ必要があります。

### 3. `null` は空文字と同じ
`null` と `""` は別物です。  
意味も扱いも異なります。

### 4. 文字と文字列は同じ
`'A'` と `"A"` は別の型です。  
混同するとエラーや設計ミスにつながります。
